# Automatic Encrypted Backups

Umbrel Lightning Node automatically backs up the channel backup file generated by LND by default. This backup helps recover Lightning channel funds if your Umbrel's storage fails, is erased, or your Lightning app data becomes corrupted.

These backups are encrypted on your Umbrel before they leave your node. Umbrel does not receive your seed phrase or unencrypted channel backup data.

Backup over Tor is enabled by default. When it is enabled, backup uploads and recovery requests are sent over Tor. If Backup over Tor is turned off, backup uploads and recovery requests are sent over clearnet instead. Backups remain encrypted either way.

Umbrel also uses random padding and occasional decoy backups to make it harder for the backup server to learn anything useful about your node or channel activity.

- Your IP address is hidden from the backup server when Backup over Tor is enabled.
- Your channel backup data is encrypted with a key derived from your Umbrel Lightning seed.
- The backup ID is derived from your seed, so a restored node can find the same backups later.
- Random padding makes backup size changes harder to infer.
- Decoy backups make it harder to correlate backup timing with channel changes.

## Turning Off Automatic Backups

Automatic backups can be turned off in the Lightning Node app settings, but this is risky. If automatic backups are off, new channel backup updates are not uploaded automatically.

If you turn this off, download and safely store a fresh channel backup every time your channels change. You can also use umbrelOS backups as another layer of protection, but make sure they include recent Lightning data before relying on them for recovery.

Turning automatic backups back on does not create a new backup identity. Your node will continue using the same seed-derived backup ID and encryption key, so previous automatic backups remain recoverable.
